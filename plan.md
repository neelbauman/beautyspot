# アイテム間ツリーグラフUI 設計・実装プラン

## UI設計

### ビューの概要: `#/tree` — インタラクティブ・トレーサビリティグラフ

D3.js を使った**レイヤード・ツリーグラフ**ビュー。ドキュメント階層（REQ → ARCH → SPEC → IMPL/TST）を左から右へレイヤーとして配置し、アイテム間のリンクをエッジで可視化する。

```

│                                                                     │
│   REQ           ARCH           SPEC          IMPL    TST           │
│  ┌─────┐      ┌─────┐       ┌─────┐       ┌─────┐ ┌─────┐       │
```

### インタラクション設計

1. **ノードの表示**
   - 各ノード = Doorstopアイテム（UID + ヘッダー短縮表示）
   - 色分け: ドキュメントprefix別（REQ=青, ARCH=紫, SPEC=緑, IMPL=橙, TST=赤）
   - ステータスアイコン: ✓(reviewed) / ○(unreviewed) / ⚠(suspect)
   - normativeでないアイテムは薄く表示

2. **エッジ（リンク）の表示**
   - 親→子のリンクを矢印付き曲線で表示
   - suspectリンクは赤い点線
   - ホバー時にリンクをハイライト

3. **リンク編集操作**
   - **リンク作成**: ノード右端のコネクタからドラッグし、別ノードにドロップ → `POST /api/items/{child}/link` で子→親リンク追加
   - **リンク削除**: エッジを右クリック → コンテキストメニューで「リンク削除」→ `POST /api/items/{child}/unlink`
   - **ノード選択**: クリックで選択 → 下部パネルに詳細 + リンク操作ボタン表示

4. **ナビゲーション**
   - マウスホイール: ズーム
   - ドラッグ(空白部分): パン
   - ダブルクリック: そのアイテムの `#/item/{uid}` を開く

5. **フィルター**
   - グループ: 選択したグループのアイテムだけ表示
   - ステータス: reviewed / unreviewed / suspect
   - 検索: UIDまたはヘッダーでフィルタ

### 下部パネル（詳細＋操作）

選択したノードの情報とリンク操作UIを表示:
- 現在の親リンク一覧（×ボタンで個別削除可能）
- 現在の子リンク一覧
- 「リンク追加」ボタン → UID入力ダイアログ or ドロップダウン

## 実装ステップ

### 1. バックエンドAPI追加 (`serve_app.py`)

新規エンドポイント:
- `GET /api/graph` — 全アイテムとリンクをグラフデータ（nodes + edges）で返す
- `POST /api/items/{uid}/link` — リンク追加 `{parent: "ARCH001"}`
- `POST /api/items/{uid}/unlink` — リンク削除 `{parent: "ARCH001"}`

`DoorstopDataStore` に追加するメソッド:
- `get_graph_data()` — `{nodes: [...], edges: [...], layers: [...]}`
- `link_item(child_uid, parent_uid)` — `item.link()` + `item.save()` + rebuild
- `unlink_item(child_uid, parent_uid)` — `item.unlink()` + `item.save()` + rebuild

### 2. フロントエンド: D3.jsロード (`spa.js`)

- CDNから D3.js v7 をロード（Mermaidと同じパターンで、タイムアウト付き）
- フォールバック: D3が使えない場合はシンプルなHTML表形式で表示

### 3. フロントエンド: ツリーグラフ描画 (`spa.js`)

- `renderTree()` 関数を追加（routerに `case 'tree'` を追加）
- SVGキャンバスをメインコンテンツ領域に生成
- レイヤード配置: ドキュメントprefixを列、アイテムを行に
- D3 force simulation でノード配置を最適化（ただしX軸はレイヤーで固定）
- ズーム・パン: `d3.zoom()`

### 4. フロントエンド: インタラクション (`spa.js`)

- ノードクリック → 選択状態 + 下部パネル表示
- ドラッグ操作でリンク作成（d3.drag + 一時的なライン表示）
- エッジ右クリック → コンテキストメニュー
- ダブルクリック → アイテム詳細へ遷移

### 5. フロントエンド: スタイリング (`spa.css`)

- ノードの色分け・ステータス表示
- エッジのスタイル（通常/suspect）
- 下部パネルのレイアウト
- コンテキストメニュー

### 6. サイドバーにナビゲーションリンク追加

- 「ツリーグラフ」リンクをサイドバーに追加
