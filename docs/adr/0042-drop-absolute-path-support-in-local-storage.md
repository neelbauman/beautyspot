# ADR 0042: Drop Absolute Path Support in LocalStorage

## Context (背景)
v1.x系の `LocalStorage` では、DB内に保存されたファイルパス（`location`）として絶対パスを許容・解決する挙動が含まれていました。v2.0 の開発において、パストラバーサル脆弱性を防ぐために `Path.is_relative_to(base_dir)` による厳密なセキュリティチェックを導入しましたが、これが旧来の絶対パスの挙動と競合し、意図せぬクラッシュ（`ValueError`）を引き起こす可能性がありました。

## Decision (決定事項)
`LocalStorage.load()` および関連するファイルアクセスにおいて、v1.x時代の絶対パスによる後方互換性の維持を公式に **放棄** します。
すべての `location` パラメータは `base_dir` に対する相対パスとしてのみ解釈され、絶対パスと解釈される文字列が渡された場合でも `base_dir` のサブディレクトリ内に解決されない限り `ValueError` として処理します。

## Rationale (理由)
1. **セキュリティの確実性**: 後方互換性のために「絶対パスの場合は許容する」という例外ルールを設けると、悪意のあるパス文字列によるディレクトリトラバーサル攻撃のベクトルを残すリスクがあります。
2. **コードのシンプルさと保守性**: 例外的なパス解決ロジックを排除することで、ストレージ層の責務が単純化されます。

## Consequences (結果)
* **Positive**: ストレージ層の堅牢性が向上し、予期せぬディレクトリへのアクセスリスクが排除されます。
* **Negative**: v1.x からの移行ユーザーで、DBの `result_value` に絶対パスが記録されている場合、キャッシュの読み込みに失敗します。
* **Mitigation**: この変更はv2系のメジャーリリースにおける破壊的変更としてリリースノートに明記します。ユーザーには、移行時にキャッシュをクリアする（`beautyspot gc`等）か、必要に応じてDBのマイグレーションを手動で行うことを推奨します。
