# ADR 0047: 実行中タスクの強参照セットによる追跡

## コンテキスト
`Spot` インスタンスは、バックグラウンドで実行中の保存タスク（`Future`）を `_active_futures` というセットで管理しています。これはシャットダウン時や `flush()` 呼び出し時に、すべてのタスクが完了したかを確認するために必要です。

## 意思決定
`_active_futures` に `weakref.WeakSet` を使用せず、通常の `set`（強参照）を使用し、コールバックによって明示的に削除する現在の実装を維持することを決定しました。

## 根拠
1. **タスクの生存保証（Durability）**: ユーザーコードが `Future` オブジェクトへの参照を即座に破棄した場合でも、`Spot` が強参照を保持することでタスクが GC されず、確実に完了まで実行されることを保証します。
2. **待機可能性（Waitability）**: `flush()` を呼び出した際、バックグラウンドループで何らかの異常が発生していても、タスクが物理的に残っている限りそれを検知できます。`WeakSet` では GC のタイミングによって「待機すべきタスクが勝手に消える」可能性があり、不透明な挙動に繋がります。
3. **決定論的な挙動**: セットからタスクが削除されるタイミングが、GC という非決定的なイベントではなく「タスク完了（callback）」という明確なライフサイクルイベントに紐付くため、デバッグやテストが容易になります。

## トレードオフ
- **メモリリークの潜在的リスク**: 何らかの理由で `add_done_callback` が一切呼ばれない極端なバグ（Python インタプリタレベルの異常など）が発生した場合、セットに参照が残り続け、メモリを占有する可能性があります。
- **対策**: プロジェクトでは `atexit` および `weakref.finalize` によるシャットダウン機構を徹底しており、プロセス終了時またはインスタンス破棄時にこれらのリソースを明示的にクリーンアップする設計をとっているため、実用上のリスクは無視できると判断しました。
