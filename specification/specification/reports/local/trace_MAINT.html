REQ010 REQ MAINT ✓ レビュー済

期限切れキャッシュの削除、孤立Blobファイルのガベージコレクション、時間ベースの一括削除（prune）等のメンテナンス操作ができること。

子: ARCH010

ARCH010 ARCH MAINT ✓ レビュー済

MaintenanceService をファサードとして、期限切れキャッシュの削除・孤立Blobの回収（GC）・統計取得等のメンテナンス操作を集約する。

DBとストレージの両方を横断的に操作し、GC時はDBに参照されていないBlobファイルを安全に削除する。レースコンディション防止のため、Blobの最終更新時刻を確認してから削除する。

親: REQ010

子: SPEC020

SPEC020 SPEC MAINT ✓ レビュー済

MaintenanceService は DB・ストレージ・シリアライザを統合したメンテナンスサービス。 clean_garbage(): DB flush → 期限切れ削除 → 孤立Blob スキャン → 削除 → 一時ファイル清掃 → 空ディレクトリ削除。 scan_garbage(grace_period): DB に参照のないBlobファイルを列挙。grace_period でインフライト保存との競合を防止。 prune(days, func_name): 指定日数以上古いキャッシュを削除。 clear(func_name): 全キャッシュまたは指定関数のキャッシュを削除。 delete_task(cache_key): DB レコードを先に削除し、次にBlobを削除（安全な削除順序）。 from_path() ファクトリでDBパスからサービスを組み立て可能。 scan_orphan_projects(): .beautyspot/blobs/ 内の孤立プロジェクトディレクトリを検出。 自動破棄 (eviction_rate): 確率的にGCをバックグラウンドで実行（最小間隔60秒）。

親: ARCH010

子: IMPL020, TST020
