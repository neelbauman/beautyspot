REQ013 REQ HERD ✓ レビュー済

同一キャッシュキーに対する並行リクエストを直列化し、1つのリクエストのみが関数を実行し、他のリクエストはその結果を共有することで重複実行を防止できること。

親: —

子: ARCH013

ARCH013 ARCH HERD ✓ レビュー済

同一キャッシュキーへの並行リクエストをイベント (threading.Event) ベースで直列化する。

最初のリクエストのみが関数を実行し、後続リクエストはイベント待機後に最初のリクエストの結果を共有する。タイムアウト・リトライ機構により、デッドロックや無限待機を防止する。

親: REQ013

子: SPEC023

SPEC023 SPEC HERD ✓ レビュー済

CacheManager 内の _inflight 辞書で同一キャッシュキーへの並行アクセスを管理する。 最初のリクエストが実行者（executor）となり関数を実行する。 後続リクエストは threading.Event (同期) または asyncio.Future (非同期) で待機し、 実行者の結果を共有する。タイムアウト (HERD_TIMEOUT=300秒) とリトライ (HERD_MAX_RETRIES=3) で無限待機を防止する。 実行者が例外を送出した場合、待機側にも同じ例外を伝播する。

親: ARCH013

子: IMPL023, TST023

IMPL023 IMPL HERD ✓ レビュー済

CacheManager の Thundering Herd Protection 実装。 _inflight 辞書、threading.Event / asyncio.Future による待機、 HERD_TIMEOUT (300s) / HERD_MAX_RETRIES (3) による保護、 例外伝播を含む。

親: SPEC023

子: —
