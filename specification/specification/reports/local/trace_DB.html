REQ003 REQ DB ✓ レビュー済

キャッシュのメタデータ（関数名、入力ID、バージョン、有効期限等）をデータベースに永続化できること。

抽象インターフェースにより、DBやストレージなどのバックエンド実装を差し替え可能であること。

親: —

子: ARCH003

ARCH003 ARCH DB ✓ レビュー済

キャッシュのメタデータ（関数名、入力ID、バージョン、有効期限等）を永続化する抽象データベースインターフェース (TaskDBBase) を定義する。

抽象化により、デフォルトのSQLite実装から他のDB（PostgreSQL等）への差し替えを可能にする。メタデータDBはキャッシュの検索・管理のインデックスとして機能し、大きなデータ本体の保存はBlobストレージ層に委譲する。

親: REQ003

子: SPEC007, SPEC008

SPEC007 SPEC DB ✓ レビュー済

デフォルトのDBのバックエンド実装はSQLiteを使用する。

  • SQLiteTaskDB はSQLiteをバックエンドとし、WALモード・専用ライタースレッドで動作する。
  • 読み取りはスレッドローカルな読み取り専用接続（PRAGMA query_only=ON）で並行実行可能。
  • 書き込みは _WriteTask キューを通じてライタースレッドが直列処理する。
  • 自動マイグレーション（content_type, version, result_data, func_identifier, expires_at）を実行する。
  • Flushable/Shutdownable プロトコルに準拠し、flush/shutdownメソッドを提供する。

親: ARCH003

子: IMPL007, TST007

SPEC008 SPEC DB ✓ レビュー済

TaskDBCore プロトコルは init_schema/get/save/delete の4メソッドを定義する。 TaskDBBase 抽象基底クラスはメンテナンス系メソッドのデフォルト実装を提供する。 Maintenable プロトコルは delete_expired/prune/get_outdated_tasks/get_blob_refs/ delete_all/get_keys_start_with/get_history を定義する。 TaskDBMaintenable = TaskDBCore + Maintenable。

親: ARCH003

子: IMPL008, TST008

IMPL007 IMPL DB ✓ レビュー済

SQLiteTaskDB の実装。WAL モード、専用ライタースレッド (_writer_loop)、 スレッドローカル読み取り専用接続 (_read_connect)、 自動マイグレーション、_WriteTask キュー、flush/shutdown を含む。
