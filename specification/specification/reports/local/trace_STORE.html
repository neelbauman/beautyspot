REQ004 REQ STORE ✓ レビュー済

大きなキャッシュデータを外部ストレージ（ローカルファイルシステムやS3等）に保存できること。ストレージポリシーによりDB直接保存とBlob保存を自動判定できること。

親: —

子: SPEC009, SPEC010, SPEC011

SPEC009 SPEC STORE ✓ レビュー済

LocalStorage はローカルファイルシステムにBlobを保存する。 save() は tempfile.mkstemp + os.replace によるアトミック書き込みで、 並行書き込みやプロセスクラッシュ時のファイル破損を防止する。 load() はパストラバーサル攻撃を is_relative_to で検証する。 list_keys() は .bin ファイルを再帰的に列挙する。 clean_temp_files() で残留一時ファイル (.spot_tmp) を清掃する。 prune_empty_dirs() で空ディレクトリを再帰的に削除する。

親: REQ004

子: IMPL009, TST009

SPEC010 SPEC STORE ✓ レビュー済

S3Storage はAWS S3互換ストレージにBlobを保存する。 boto3 の upload_fileobj によるマルチパートアップロードに対応（5GB超対応）。 s3:// URI 形式でバケット名・プレフィックスを指定する。 boto3 がインストールされていない場合は ImportError を送出する。

親: REQ004

子: IMPL010, TST010

SPEC011 SPEC STORE ✓ レビュー済

StoragePolicyProtocol は should_save_as_blob(data: bytes) -> bool を定義する。 実装3種: ThresholdStoragePolicy（閾値超過時にBlob保存）、 WarningOnlyPolicy（警告のみでBlob保存しない、デフォルト）、 AlwaysBlobPolicy（常にBlob保存）。 @mark(save_blob=True/False) による明示指定が最優先。

親: REQ004

子: IMPL011, TST011

IMPL009 IMPL STORE ✓ レビュー済

LocalStorage の実装。tempfile.mkstemp + os.replace によるアトミック書き込み、 is_relative_to によるパストラバーサル防止、.bin 再帰列挙、 clean_temp_files/prune_empty_dirs を含む。
