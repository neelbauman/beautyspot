REQ002 REQ KEY ✓ レビュー済

関数の引数から安定かつ一意なキャッシュキーを生成できること。引数の型・構造に応じた正規化を行い、辞書のキー順序やコレクション型の違いを正しく区別できること。

親: —

子: SPEC004, SPEC005, SPEC006

SPEC004 SPEC KEY ✓ レビュー済

キャッシュキーは func_identifier:input_id[:version] 形式の文字列に対する SHA-256 ハッシュとして生成する。func_identifier は module.qualname 形式。 input_id はデフォルトでは引数の正規化結果を msgpack + SHA-256 でハッシュした値。

親: REQ002

子: IMPL004, TST004

SPEC005 SPEC KEY ✓ レビュー済

canonicalize() 関数（singledispatch）により、引数を再帰的に正規化する。 dict はキーでソート、set/frozenset/list/tuple/deque/defaultdict/OrderedDict/Enum は 型タグ付きで正規化し、型の違いによるキャッシュキー衝突を防ぐ。 numpy 配列は shape+dtype+tobytes で正規化。カスタムオブジェクトは dict/slots を使用。 bool と int の区別（True vs 1）を保証する。

親: REQ002

子: IMPL005, TST005

SPEC006 SPEC KEY ✓ レビュー済

KeyGen.ignore(*arg_names), KeyGen.map(**strategies), KeyGen.file_content(*args), KeyGen.path_stat(*args) のファクトリメソッドにより、引数ごとのキー生成戦略を指定する KeyGenPolicy を生成できる。Strategy は DEFAULT, IGNORE, FILE_CONTENT, PATH_STAT の4種。 KeyGenPolicy.bind(func) で関数シグネチャにバインドされたキー生成関数を返す。

親: REQ002

子: IMPL006, TST006
